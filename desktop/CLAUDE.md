# Hitch Desktop — CLAUDE.md

## Design Context

This app has captured design context for the `impeccable` design skill:

- **[PRODUCT.md](./PRODUCT.md)** — strategic: register (product), platform (web), users, purpose, positioning, personality, anti-references, principles.
- **[DESIGN.md](./DESIGN.md)** — visual: the monochrome token system, type scale, elevation, components, and do's/don'ts. Tokens are normative.

Read both before designing or changing UI here. In short:

- **Positioning:** Hitch enables the effortless expression of developer intent — every screen shortens the distance between what a developer means and what their agents do with it.
- **Feel:** warm, human, effortless — but warmth comes from voice, timing, and space, **not** color. The North Star is *"The Clean Desk": obvious, calm, inviting.*
- **Visual doctrine:** monochrome chrome (zero-chroma OKLCH ramp, light + dark). Color is information, reserved for user-assigned tag tints only. Flat by default; hairline borders and tonal layering, not shadows and cards. One sans family; 26px working-title ceiling.
- **Never:** busy SaaS dashboard, AI-product cliché (violet gradients / sparkles / magic borders), Jira-style ceremony, or anything that reads as overwhelming.

Run `/impeccable` (no argument) for the context-aware menu of next design commands.
